========================================
 FRESH GITHUB SETUP - COMPLETE GUIDE
========================================

PROBLEM:
- STS (Spring Tool Suite) created a .git folder in Backend
- This prevented Backend code from being pushed
- Old repository has this issue

SOLUTION:
- Delete old repository on GitHub
- Create NEW empty repository
- Use the setup script for fresh push

========================================
 STEP-BY-STEP INSTRUCTIONS
========================================

STEP 1: DELETE OLD REPOSITORY ON GITHUB
----------------------------------------
1. Go to: https://github.com/KishanRaghav/Jovac-Project
2. Click "Settings" (at the top)
3. Scroll to bottom
4. Click "Delete this repository"
5. Type the repository name to confirm
6. Click "I understand, delete this repository"

STEP 2: CREATE NEW EMPTY REPOSITORY
------------------------------------
1. Go to: https://github.com/new
2. Repository name: Jovac-Project (or any name)
3. Description: Student Management System
4. Choose: Public or Private
5. DO NOT initialize with README
6. DO NOT add .gitignore
7. DO NOT add license
8. Click "Create repository"
9. COPY the repository URL shown

STEP 3: RUN THE SETUP SCRIPT
-----------------------------
1. Double-click: SETUP-FRESH-GITHUB.bat
2. Press any key to start
3. Wait for cleanup to complete
4. Paste your NEW repository URL when asked
5. Type 'PUSH' to confirm
6. Wait for upload to complete

STEP 4: VERIFY ON GITHUB
-------------------------
1. Visit your repository on GitHub
2. Check that ALL folders are visible:
   - Backend/Student-Management-System/src/
   - Frontend/student-management-frontend/src/
   - All .bat scripts
   - README.md
